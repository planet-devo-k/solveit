<h1 align="center">SOLVEIT</h1>
<p align="center">우리의 답은 하나가 아니야 — 코딩 테스트, 우리 함께 Solve It!</p>

## Introduction

solveit은 함께 코딩 테스트 문제를 풀고, 리뷰와 피드백을 통해 실력을 키우는 스터디입니다.

📌 **진행 상황**은 [Milestones](https://github.com/front-studium/solveit/milestones)와 [Roadmap](https://github.com/orgs/front-studium/projects/1/views/1)을 참고하세요.

- **기간**: 2025.04.03 ~ ING
- **시간**: 매주 일요일 오후 10시

- **목표**: 다양한 문제를 해결하고, 피드백을 통해 실력을 향상시키는 것을 목표로 합니다.
- **내용**: 매주 코딩테스트 2-3문제를 풉니다.

- **언어**: **JavaScript**

## Rules

**Ground Rules**

- [PLANET DEVO K GROUND RULES](https://github.com/planet-devo-k#ground-rules)

**Branch & Directory**

- 각 참여자는 자신의 username으로 브랜치를 만들어 개별적으로 코딩합니다.
- 각 참여자는 자신의 username으로 폴더를 만들고 그 안에서 작업합니다.

**PR**

- 매주 **토요일 자정**까지 `main` 브랜치로 PR을 제출합니다.
- 매주 **일요일 오후8시**까지 코드 리뷰를 남깁니다.
- 각 PR에는 최소 2명의 코드 리뷰가 필수입니다. 
- 리뷰어는 커스텀 로직으로 자동 배정됩니다.
- PR 병합은 2명 이상의 승인(Approve)이 있어야 가능합니다.

**Issue**

- 어려운 문제를 만났을 경우, Discussions탭의 [Q&A](https://github.com/front-studium/solveit/discussions/categories/q-a)를 활용해 질문 및 해결 방법을 공유합니다.
- 다음과 같은 라벨을 사용합니다:
  - **카테고리**: `array`, `string` 등
  - **난이도**: `level 0`, `level 1` 등
  - **출처**: `Programmers` 등

**Commit**

- 하나의 문제를 해결한 후 커밋합니다.
- 아래의 커밋 메시지 작성 규칙을 따릅니다:

| 타입        | 메시지                      | 예시                                                              |
| ----------- | --------------------------- | ----------------------------------------------------------------- |
| `solve:`    | 문제를 해결했을 때 사용     | `solve: merge strings alternately`                                |
| `fix:`      | 버그를 수정했을 때 사용     | `fix: correct logic for checking the greatest common divisor`     |
| `optimize:` | 코드 최적화                 | `optimize: simplify the algorithm for finding the common divisor` |
| `note:`     | 설명 또는 노트 추가 시 사용 | `note: add explanation for 'merge strings alternately' solution`  |
| `docs:`     | 문서 업데이트 시 사용       | `docs: update README with explanation for new problem solution`   |
| `test:`     | 테스트 추가 시 사용         | `test: add notifier workflow `                                    |
| `chore:`    | 기타 변경 사항              | `chore: update package dependencies`                              |
